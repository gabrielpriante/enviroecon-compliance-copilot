# Example Compliance Memoranda and Usage Examples

This directory will contain example compliance memoranda and usage scenarios demonstrating the EnviroEcon Compliance Copilot system in action.

## Planned Examples

### 1. Hazardous Waste Determination
**Scenario**: Municipal facility needs to determine if waste stream is hazardous
**Demonstrates**: 
- Citation-first analysis
- Conservative interpretation
- Explicit assumptions
- Professional review recommendations

### 2. NPDES Permit Applicability
**Scenario**: Nonprofit evaluating discharge permit requirements
**Demonstrates**:
- Jurisdictional analysis
- Multi-regulation synthesis
- Uncertainty flagging
- Alternative interpretations

### 3. CERCLA Liability Assessment
**Scenario**: Consultant researching potential responsible party status
**Demonstrates**:
- Complex regulatory framework navigation
- Case law integration
- Defensive analysis approach
- Clear limitation disclosure

### 4. State vs. Federal Compliance
**Scenario**: Comparing state and federal air quality requirements
**Demonstrates**:
- Multi-jurisdictional analysis
- Incorporated-by-reference handling
- Stricter standard identification
- Complete citation chain

## Example Structure

Each example will include:

```
example_name/
├── query.md              # Original compliance question
├── memorandum.md         # Generated compliance memorandum
├── assumptions.md        # Detailed assumption documentation
├── audit_trail.md        # Complete source lineage
└── professional_notes.md # Notes for professional review
```

## Usage Guidelines

These examples are:
- ✅ Educational demonstrations of system capabilities
- ✅ Templates for expected output quality
- ✅ Test cases for system validation
- ❌ Not actual legal advice
- ❌ Not applicable without professional review
- ❌ Not substitutes for site-specific analysis

## Contributing Examples

When adding examples:
1. Use realistic but anonymized scenarios
2. Demonstrate proper guardrail application
3. Include complete citation verification
4. Show assumption tracking
5. Include all output components (memorandum, assumptions, audit trail)
6. Add professional review recommendations

*Examples are for demonstration purposes only and should not be relied upon for actual compliance decisions.*
